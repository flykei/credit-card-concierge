# プロジェクト概要：クレカコンシェルジュ

## 🎯 プロジェクトの目的

ユーザーの年間支出額と内訳を入力することで、実質還元額（獲得ポイント - 年会費）が最大になるクレジットカードを提案するWebツール。

## ✅ 完成状況

**すべての機能が実装完了し、動作確認済みです！**

### 実装済み機能

1. ✅ **支出入力フォーム**
   - 年間支出合計額の入力
   - 支出内訳の詳細入力（Amazon、楽天、コンビニなど14カテゴリ）
   - バリデーション機能

2. ✅ **還元額計算エンジン**
   - 基本還元率の計算
   - カテゴリ別ボーナス還元の計算
   - 年間利用額ボーナスの計算
   - 実質還元額 = 獲得ポイント - 年会費

3. ✅ **結果表示UI**
   - ランキング形式での表示
   - 詳細内訳の展開表示
   - レスポンシブデザイン

4. ✅ **SEO対策**
   - メタデータ最適化（タイトル、ディスクリプション、OGP）
   - 構造化データ（JSON-LD: WebApplication, FAQPage）
   - サイトマップ生成
   - robots.txt生成

5. ✅ **対応カード（10種類）**
   - 三井住友カード プラチナプリファード
   - 三井住友カード ゴールド(NL)
   - ジャックスカードプラチナ
   - エポスプラチナカード
   - リクルートカード
   - P-oneカード Wiz
   - メルカード ゴールド
   - Marriott Bonvoy アメックス・プレミアム
   - 住信SBIネット銀行 プラチナデビットカード
   - 住信SBIネット銀行 デビットカード Point+

## 🔧 カスタマイズのポイント

### カードの追加方法

1. `data/cards.json` に新しいカード情報を追加
   \`\`\`json
   {
     "id": "new-card-id",
     "name": "新しいカード名",
     "issuer": "発行会社",
     "annualFee": 11000,
     "baseRate": 1.0,
     "bonusRates": [...],
     "specialBonuses": [...]
   }
   \`\`\`

2. `app/page.tsx` の対応カード一覧を更新

### 支出カテゴリの追加

1. `lib/types.ts` の `MERCHANT_CATEGORIES` に追加
2. `components/ExpenseInput.tsx` でUI調整（必要に応じて）

### デザインのカスタマイズ

- Tailwind CSSを使用しているため、クラス名を変更するだけで簡単にカスタマイズ可能
- カラースキーム: `app/globals.css` で調整

## ⚠️ 注意事項

1. **カード情報の更新**
   - 還元率や年会費は変更される可能性があります
